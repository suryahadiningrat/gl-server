1. Tujuan Dokumen

Dokumen ini disusun sebagai panduan operasional untuk mengelola fitur Citra Drone pada Sistem Mandara. Panduan ini mencakup langkah-langkah akses VPN, akses server, penanganan gangguan (troubleshooting), dan cara mengunggah data citra drone secara massal.

Dokumen ini ditujukan kepada tim teknis (IT Staff) Kementerian Lingkungan Hidup dan Kehutanan yang bertanggung jawab atas operasional dan pemeliharaan sistem.

5. Troubleshooting Fitur Citra Drone

Apabila peta citra drone tidak tampil atau data tidak diperbarui, lakukan langkah-langkah berikut dari dalam server (setelah login SSH).

5.1 Restart Container TileServer

Container tileserver-zurich bertanggung jawab menyajikan tile peta citra drone. Apabila peta tidak muncul atau tileserver tidak merespons, lakukan restart:

  1. Masuk ke server via SSH (lihat Bagian 5).
  2. Pindah ke folder gl-server:
cd /home/femandara/docker/gl-server
  1. Restart container tileserver:
docker compose restart tileserver-zurich
  1. Tunggu beberapa detik, lalu verifikasi container kembali aktif:
docker ps | grep tileserver
  1. Apabila container tidak muncul atau statusnya bukan Up, jalankan perintah berikut untuk melihat log error:
docker logs --tail=50 tileserver-zurich
Catatan: Setelah restart, tileserver membutuhkan waktu sekitar 10–30 detik untuk siap melayani permintaan tile. Muat ulang halaman peta di browser setelah container aktif.

5.2 Menjalankan Script generate-config.sh

Script generate-config.sh digunakan untuk memperbarui file konfigurasi tileserver (config.json) apabila ada data MBTiles baru yang ditambahkan secara manual ke server, atau apabila konfigurasi tileserver perlu diregenerasi dari awal.

Perhatian: Script ini akan menimpa file config.json yang ada. Gunakan hanya apabila diperlukan, misalnya setelah penambahan file MBTiles baru secara manual atau setelah terjadi kerusakan pada konfigurasi.
  1. Masuk ke server via SSH dan berpindah ke folder gl-server:
cd /home/femandara/docker/gl-server
  1. Jalankan script generate-config:
bash generate-config.sh

Atau apabila perlu melakukan rebuild penuh (memproses semua file dari awal):

FORCE_REBUILD=true bash generate-config.sh
  1. Setelah script selesai, restart tileserver agar konfigurasi baru diterapkan:
docker compose restart tileserver-zurich
  1. Verifikasi konfigurasi berhasil diperbarui:
cat config.json | python3 -m json.tool | grep -c "sources"

Angka yang muncul menunjukkan jumlah dataset yang terdaftar dalam konfigurasi tileserver.

6. Bulk Upload Citra Drone (MBTiles)

Script bulk-upload-mbtiles.sh digunakan untuk mengunggah banyak file citra drone (.mbtiles) sekaligus ke sistem. Script secara otomatis:

  • Mengunggah setiap file ke penyimpanan (MinIO)
  • Mendaftarkan metadata ke database
  • Memperbarui konfigurasi tileserver
  • Melewati file yang sudah pernah diunggah sebelumnya (tidak upload ulang)

6.1 Prasyarat

Pastikan perangkat lunak berikut sudah terinstal sebelum menjalankan script:

DependensiFungsiInstalasi (Windows)Instalasi (Linux/Mac)
Git Bash Terminal Bash untuk menjalankan script di Windows Unduh dari git-scm.com, instal dengan opsi default Sudah tersedia (gunakan Terminal biasa)
mc (MinIO Client) Upload file ke MinIO storage Unduh mc.exe dari min.io, letakkan di folder yang ada di PATH brew install minio/stable/mc
psql Koneksi ke database PostgreSQL Instal PostgreSQL dari postgresql.org (sudah termasuk psql) sudo apt install postgresql-client
sqlite3 Validasi file MBTiles Unduh dari sqlite.org dan tambahkan ke PATH Biasanya sudah tersedia, atau apt install sqlite3
Catatan Windows: Script harus dijalankan melalui Git Bash, bukan Command Prompt atau PowerShell biasa. Setelah instal Git Bash, klik kanan di folder tempat script berada, pilih "Git Bash Here".

6.2 Cara Menjalankan di Windows (Git Bash)

  1. Pastikan semua prasyarat sudah terinstal (lihat 7.1).
  2. Salin script bulk-upload-mbtiles.sh ke komputer lokal (lihat Lampiran A).
  3. Buka folder tempat script berada, klik kanan, pilih Git Bash Here.
  4. Beri izin eksekusi pada script:
chmod +x bulk-upload-mbtiles.sh
  1. Jalankan script dengan perintah berikut (sesuaikan path folder dan parameter):
./bulk-upload-mbtiles.sh "C:/Users/NamaAnda/Documents/Citra Drone/" \
  --tahun 2026 --bulan 6 --status "Aktif" 2>&1 | tee logs.log
Catatan: Apabila path folder mengandung spasi, pastikan diapit tanda kutip ganda ("). Perintah tee logs.log akan menyimpan seluruh output ke file logs.log di folder yang sama.

6.3 Cara Menjalankan di Linux / Dari Server

  1. Login ke server via SSH (lihat Bagian 5).
  2. Pindah ke folder gl-server:
cd /home/femandara/docker/gl-server
  1. Pastikan script executable:
chmod +x bulk-upload-mbtiles.sh
  1. Jalankan script:
./bulk-upload-mbtiles.sh "/path/ke/folder/mbtiles/" \
  --tahun 2026 --bulan 6 --status "Aktif" 2>&1 | tee logs.log

6.5 Contoh Perintah

Upload biasa (tahun + bulan):

./bulk-upload-mbtiles.sh "/Data/Citra Drone/Juni 2026/" \
  --tahun 2026 --bulan 6 --status "Aktif" 2>&1 | tee logs.log

Upload dengan BPDAS spesifik:

./bulk-upload-mbtiles.sh "/Data/mbtiles/" \
  --tahun 2026 --bulan 6 --bpdas "BPDASMAHAKAMBERAU" \
  --status "Aktif" --operator "Tim KLHK" 2>&1 | tee logs.log

Dry-run (cek tanpa upload):

./bulk-upload-mbtiles.sh "/Data/mbtiles/" --tahun 2026 --bulan 6 --dry-run

Melihat ringkasan error dari log:

grep "[ERROR]" logs.log
